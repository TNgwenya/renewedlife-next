# GitHub Copilot / ChatGPT 5.4 Execution Brief

## Objective
Take the existing Next.js prototype for **renewedlifeint.com** and complete it into a polished, production-ready, mobile-first church website.

## Role
Act as:
- senior product designer
- senior frontend engineer
- UX strategist
- content implementer
- QA reviewer

## Non-negotiable outcomes
- modern, premium-feeling church website
- first-time visitor journey must be obvious
- mobile experience must be excellent
- strong homepage CTA flow
- easy content maintenance
- clear visual hierarchy
- faithful to Renewed Life's identity

## Stack direction
- Next.js
- reusable component architecture
- responsive design
- clean section-based page composition
- simple maintainable styling system
- production-ready code quality

## Priority pages
1. Home
2. Plan Your Visit
3. About
4. Ministries
5. Sermons
6. Give
7. Contact
8. Events (optional or phase 2)

## Immediate build tasks
1. Audit the current prototype structure.
2. Refine the route/page architecture.
3. Create a polished design system:
   - typography scale
   - spacing system
   - button variants
   - card variants
   - section shells
4. Rebuild homepage based on the benchmark findings.
5. Build a proper Plan Your Visit page.
6. Strengthen About, Ministries, Sermons, Give, and Contact pages.
7. Add reusable CTA sections across pages.
8. Prepare integration placeholders for:
   - YouTube
   - WhatsApp
   - Google Maps
   - online giving
9. Add clean footer and contact utility structure.
10. Perform responsive QA.

## UX rules
- Never bury service time and location.
- Every major page should have a next action.
- Prioritize clarity over complexity.
- Design for first-time visitors before internal users.
- Keep text scannable.
- Avoid clutter and weak CTA competition.

## Copy rules
- warm
- pastoral
- welcoming
- modern
- concise
- clear to both Christians and non-churched visitors

## Deliverable expectation
Complete the site to a state where a human only needs to:
- review content
- add final media assets
- connect final production integrations
- deploy

## Suggested work sequence
### Step 1 — Structure
Lock sitemap, nav, footer, and route layout.

### Step 2 — Design System
Create consistent reusable UI patterns.

### Step 3 — Homepage
Build the strongest conversion-focused homepage.

### Step 4 — Plan Your Visit
Make this page exceptional.

### Step 5 — Secondary Pages
About, Ministries, Sermons, Give, Contact.

### Step 6 — Integrations
Add placeholders and production-ready hooks.

### Step 7 — QA
Mobile, tablet, desktop, spacing, readability, navigation, consistency.

## Completion standard
Do not stop at “working.”
The goal is **polished, launchable, and benchmark-aligned**.
